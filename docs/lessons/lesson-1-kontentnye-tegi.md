# Конспект | Контентные теги

## План занятия (общий для модуля) {#план-занятия-общий-для-модуля}

* Шесть уровней заголовков: `h1–h6`
* Вёрстка основного текста: `p`, `line-height`
* Вёрстка списков: `ul`, `ol`, `li`, `list-style-type`
* Изображения и подписи: `img`, `figure`, `figcaption`
* Отступы и рамки: `padding`, `margin`, `border`
* Обтекание: `float`
* Идентификаторы и классы: `id`, `class`

## Шесть уровней заголовков (h1–h6) {#шесть-уровней-заголовков-h1-h6}

Заголовки — это не просто выделенный текст, а «скелет» страницы. Они создают структуру, по которой пользователь сканирует контент, а поисковики и скринридеры понимают смысл. `h1` используется для главной темы страницы, `h2` — для разделов, `h3–h6` — для уточнений и деталей. Нарушение иерархии делает страницу нечитаемой и хаотичной.

**Правило.** Уровни отражают важность: `h1` — глобальный заголовок (один на страницу), `h2` — основные блоки, `h3–h6` — подразделы. Визуальный стиль задаётся CSS, но семантика определяется тегами.

### Исходный вариант (ошибка: нет иерархии)

```html
<h1>Пушкин, Александр Сергеевич</h1>

Биография
Происхождение
<!-- текст -->
Молодость
<!-- текст -->
Творчество Пушкина
Литературная репутация
<!-- текст -->
Произведения
Поэмы
<!-- текст -->
Сказки
<!-- текст -->
```

### Шаг 1 (частичное исправление: выделили разделы `h2`)

```html
<h1>Пушкин, Александр Сергеевич</h1>

<h2>Биография</h2>
Происхождение
<!-- текст -->
Молодость
<!-- текст -->

<h2>Творчество Пушкина</h2>
Литературная репутация
<!-- текст -->
Произведения
Поэмы
<!-- текст -->
Сказки
<!-- текст -->
```

### Шаг 2 (добавили вложенность через `h3`)

```html
<h1>Пушкин, Александр Сергеевич</h1>

<h2>Биография</h2>
  <h3>Происхождение</h3>
  <!-- текст -->
  <h3>Молодость</h3>
  <!-- текст -->

<h2>Творчество Пушкина</h2>
  <h3>Литературная репутация</h3>
  <!-- текст -->
  <h3>Произведения</h3>
  Поэмы
  <!-- текст -->
  Сказки
  <!-- текст -->
```

### Шаг 3 (полная иерархия: добавили `h4` и вынесли оформление в CSS)

```html
<h1>Пушкин, Александр Сергеевич</h1>

<h2>Биография</h2>
  <h3>Происхождение</h3>
  <!-- текст -->
  <h3>Молодость</h3>
  <!-- текст -->

<h2>Творчество Пушкина</h2>
  <h3>Литературная репутация</h3>
  <!-- текст -->
  <h3>Произведения</h3>
    <h4>Поэмы</h4>
    <!-- список -->
    <h4>Сказки</h4>
    <!-- список -->
```

```css
h1 { font-size: 32px; color: #333; }
h2 { font-size: 24px; border-bottom: 1px solid #ccc; }
h3, h4 { font-size: 18px; }
```

#### Разбор по шагам

* `h1` — корневой заголовок.
* `h2` — крупные разделы.
* `h3` — подразделы внутри `h2`.
* `h4–h6` — уточнения.
* Оформление (размер, цвет, подчёркивания) задаётся CSS, а не выбором уровня заголовка.

**Ключевые правила:**

* Один `h1` на страницу.
* Последовательность уровней без «скачков» (после `h2` идёт `h3`, а не сразу `h5`).
* Семантика важнее внешнего вида.
* Проверяйте дерево заголовков в WAVE или Lighthouse.
* Ошибка новичков: использовать `h1` везде ради «крупного текста».

## Вёрстка основного текста (`p`, `line-height`) {#вёрстка-основного-текста-p-line-height}

Основной текст — это «тело» страницы. Тег `p` группирует предложения в абзацы, браузер автоматически добавляет отступы (`margin`). Свойство `line-height` регулирует межстрочный интервал, влияя на читаемость.

**Правило.** `p` используется для смысловых блоков текста. `line-height` лучше задавать в относительных единицах (`1.5–1.8`), чтобы текст был адаптивным.

### Пример разметки абзацев

```html
<p>Текст — это письменная или звучащая речь, которая внутренне организована и относительно закончена.</p>

<p>В веб‑разработке абзацы помогают разбивать «стену текста» на удобные куски.</p>
```

### Настройка межстрочного интервала

```css
p {
  line-height: 1.6;  /* Оптимальное значение для экранов */
}

p.tight {
  line-height: 1.2;  /* Более плотный текст */
}

p.loose {
  line-height: 2;    /* Более «воздушный» текст */
}
```

#### Разбор по шагам

* Каждый смысловой блок — отдельный `p`.
* Отступы делаются через CSS, а не пустыми тегами.
* `line-height` подбирается под размер шрифта и тип контента.

**Ключевые правила:**

* Не вкладывайте `p` в `p` (недопустимо в HTML).
* Для интервалов используйте `margin`, а не пустые строки.
* Для читаемости используйте короткие абзацы (3–5 строк).
* Ошибка новичков: вставлять `<br>` вместо `p`.

## Вёрстка списков (`ul`, `ol`, `li`, `list-style-type`) {#вёрстка-списков-ul-ol-li-list-style-type}

Списки упрощают восприятие информации: пошаговые инструкции, преимущества, меню. Выбор `ol` или `ul` зависит от того, важен ли порядок.

**Правило.** `li` используется только внутри `ul` или `ol`. Свойство `list-style-type` задаёт вид маркеров или нумерации.

### Упорядоченный список (`ol`)

```html
<p>Я жил в следующих странах (в порядке проживания):</p>

<ol>
  <li>Швейцария</li>
  <li>Великобритания</li>
  <li>США</li>
  <li>Норвегия</li>
</ol>
```

### Неупорядоченный список (`ul`)

```html
<p>Я жил в следующих странах:</p>

<ul>
  <li>Норвегия</li>
  <li>Великобритания</li>
  <li>США</li>
  <li>Швейцария</li>
</ul>
```

### Типы маркеров/нумерации (`list-style-type`)

```css
/* Маркеры для ul */
ul { list-style-type: disc; }             /* ● (по умолчанию) */
ul.circle { list-style-type: circle; }    /* ○ */
ul.square { list-style-type: square; }    /* ■ */
ul.none { list-style-type: none; }        /* без маркера */

/* Нумерация для ol */
ol { list-style-type: decimal; }                       /* 1, 2, 3 */
ol.zero { list-style-type: decimal-leading-zero; }     /* 01, 02, 03 */
ol.lower-alpha { list-style-type: lower-alpha; }       /* a, b, c */
ol.upper-alpha { list-style-type: upper-alpha; }       /* A, B, C */
ol.lower-roman { list-style-type: lower-roman; }       /* i, ii, iii */
ol.upper-roman { list-style-type: upper-roman; }       /* I, II, III */
ol.lower-greek { list-style-type: lower-greek; }       /* α, β, γ */
ol.lower-latin { list-style-type: lower-latin; }       /* a, b, c */
ol.upper-latin { list-style-type: upper-latin; }       /* A, B, C */
ol.armenian { list-style-type: armenian; }
ol.georgian { list-style-type: georgian; }
```

### Пример

```html
<ol class="zero">
  <li>Вена, Австрия</li>
  <li>Мельбурн, Австралия</li>
  <li>Осака, Япония</li>
</ol>

<ol class="lower-alpha">
  <li>Вена, Австрия</li>
  <li>Мельбурн, Австралия</li>
  <li>Осака, Япония</li>
</ol>

<ul class="circle">
  <li>Вена, Австрия
    <ul class="square">
      <li>Население: 1.9 млн</li>
      <li>Достопримечательности: Опера, Шёнбрунн</li>
    </ul>
  </li>
  <li>Мельбурн, Австралия</li>
  <li>Осака, Япония</li>
</ul>
```

**Ключевые правила:**

* `li` используется только внутри `ul` или `ol`.
* Вложенные списки помещаются внутрь `li`.
* `list-style-type: none;` + псевдоэлементы (`::before`) — для кастомных маркеров.
* Для табличных данных используйте `table`, а не списки.

## Изображения и подписи (`img`, `figure`, `figcaption`) {#изображения-и-подписи-img-figure-figcaption}

### Атрибуты `img`

* `src` — путь к файлу/URL.
* `alt` — альтернативный текст (важен для доступности).

```html
<img src="shalott.jpeg" alt="Молодая девушка в лодке">  <!-- содержательный alt -->
<img src="piechart.gif" alt="Круговая диаграмма: Chrome 25%, Firefox 40%, IE 25%, Safari 6%, Opera 4%">  <!-- график -->
<img src="divider.svg" alt="">  <!-- декоративное изображение -->
```

### `figure` + `figcaption`

```html
<figure>
  <img src="sobor.jpg" alt="Успенский собор, белый камень, золотой купол">
  <figcaption>Успенский собор во Владимире, XII век</figcaption>
</figure>

<figure>
  <img src="sobor.jpg" alt="Успенский собор">
  <figcaption>
    <p>Успенский собор во Владимире, XII век.</p>
    <p>Автор проекта: архитектурная справка.</p>
  </figcaption>
</figure>
```

**Ключевые правила:**

* Заполняйте `alt` по смыслу; для декоративных изображений используйте пустой `alt`.
* В `figure` допускается один `figcaption`.
* Избегайте использования изображений как текста (текст должен быть в HTML).

## Стилизация: свойство `float` {#стилизация-свойство-float}

```html
<img class="left" src="author.jpg" alt="Портрет автора">
<p>Текст, обтекающий изображение.</p>
```

```css
img.left {
  float: left;
  margin: 0 16px 8px 0;
  width: 200px;
}
img.right {
  float: right;
  margin: 0 0 8px 16px;
  width: 200px;
}
.clearfix::after { content: ""; display: block; clear: both; }
```

**Ключевые правила:**

* `float` выносит элемент в поток текста (left/right).
* Для современных макетов предпочтительнее Flexbox/Grid; `float` применяется для обтекания мелких медиа‑блоков.

## Отступы и рамки (`padding`, `margin`, `border`) {#отступы-и-рамки-padding-margin-border}

### `padding` (внутренние отступы)

```css
.box-a {
  padding-top: 15px;
  padding-right: 10px;
  padding-bottom: 20px;
  padding-left: 0;
}

.box-1 { padding: 30px; }
.box-2 { padding: 40px 30px; }
.box-3 { padding: 40px 30px 35px; }
.box-4 { padding: 40px 30px 35px 50px; }
```

### `border` (рамка)

```css
.b-width {
  border-top-width: 2px;
  border-right-width: 1px;
  border-bottom-width: 3px;
  border-left-width: 1px;
}

.b-style-1 { border-style: solid; }
.b-style-2 { border-style: dotted; }
.b-style-3 { border-style: dashed; }
.b-style-4 { border-style: double; }

.b-color {
  border-top-color: red;
  border-right-color: black;
  border-bottom-color: gray;
  border-left-color: blue;
}

.b-short { border: 1px dotted #000; }
```

### Комбинированный пример

```html
<div class="card">
  <h4>Заголовок карточки</h4>
  <p>Содержимое карточки с отступами и рамкой.</p>
</div>
```

```css
.card {
  padding: 16px 20px;
  border: 1px solid #ddd;
  margin-bottom: 16px;
  border-radius: 6px;
  background: #fff;
}
```

**Ключевые правила:**

- `padding` добавляет «воздух» внутри рамки, `margin` — снаружи.
- Шорткаты читаемее, но помните порядок значений: top, right, bottom, left.
- Для видимой рамки всегда указывайте и `border-style`, и `border-width`, и `border-color` (или шорткат `border`).
- Проверяйте интерактивные области: достаточный `padding` делает кнопки и ссылки удобнее.

## Внешние отступы и строчные контейнеры (`margin`, `span`)

### Свойство `margin`

```css
/* Отдельные стороны */
margin-top: 40px;
margin-right: 50px;
margin-bottom: 35px;
margin-left: 30px;

/* Короткие записи */
margin: 40px 30px 35px 50px;  /* top right bottom left */
margin: 40px 30px 35px;       /* top left/right bottom */
margin: 40px 30px;            /* top/bottom left/right */
margin: 30px;                 /* Все стороны одинаково */
```

```html
<div style="margin: 30px; border: 1px solid #ccc;">
  Lorem ipsum dolor sit amet, consectetur adipiscing elit.
</div>
```

#### Пример в контексте (с `figure`)

```css
figure {
  float: left;
  padding: 15px;
  border: 2px solid #ddd;
  margin: 40px 30px 35px 50px;
}

figcaption {
  font-size: 13px;
}
```

### Тег `span`

```html
<p>Выше представлены <span>сервисы Яндекса</span></p>
```

```css
span {
  font-weight: bold;
  color: blue;
}
```

#### Правила

* `margin` задаёт внешние отступы, не влияя на размеры контента.
* Допустимы отрицательные значения, но использовать осторожно.
* Вертикальные `margin` могут коллапсировать: берётся большее из двух.
* `span` — строчный контейнер, не принимает размеры (`width`, `height`), для этого нужен `display: inline-block;`.
* Для семантического выделения использовать `em` и `strong`, `span` — только для стиля.

## Идентификатор и класс (`id`, `class`)

### Пример с `id`

```html
<ul id="yandex-services">
  <li><a href="#">Яндекс Видео</a></li>
  <li><a href="#">Яндекс Картинки</a></li>
</ul>
```

```css
#yandex-services {
  line-height: 24px;
}
```

Ошибки:

```html
<!-- Дубликат id -->
<p id="yandex-services">Выше представлены сервисы Яндекса</p>

<!-- Несколько значений недопустимы -->
<ul id="yandex-services list">
  <li>...</li>
</ul>
```

### Пример с `class`

```html
<ul class="yandex-services list">
  <li><a href="#" class="decoration-text">Яндекс Видео</a></li>
  <li><a href="#" class="decoration-text">Яндекс Картинки</a></li>
</ul>
```

```css
.yandex-services {
  line-height: 24px;
}

.list {
  margin-top: 20px;
}

.decoration-text {
  font-weight: 700;
}
```

#### Правила

* `id` — уникален на странице, применяется для одного элемента.
* `class` — можно указывать несколько, стили суммируются.
* `#id` имеет более высокую специфичность, чем `.class`.
* Для стилей используйте классы, для JS и якорей — идентификаторы.

## Итоги

* Контентные теги (`h1–h6`, `p`, `ul/ol/li`, `img`, `figure/figcaption`, `span`) формируют структуру.
* Свойства (`line-height`, `list-style-type`, `float`, `padding`, `border`, `margin`) управляют отображением.
* Селекторы (`id`, `class`) помогают назначать стили: `id` — для уникальных элементов, `class` — для повторных.
* Практика: Соберите макет с 10+ тегами, протестируйте читаемость на мобильных устройствах.
